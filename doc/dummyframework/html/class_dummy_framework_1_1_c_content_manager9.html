Detailed Description

Basic content manager.

The content manager keeps track of resources, and handles them on device loss/reset. However it does not make any reference counting on them, so if you no longer use a resource that will be released only when the program terminates.

Resources in the default pool are handled specially (f. e. render targets). You can query the recreated resource through the GetTextureByID() method, when the device is reset.

Note that the manager handles resources only created through its methods, thus if you get a texture's surface level for example, you have to release that on device loss/termination.
