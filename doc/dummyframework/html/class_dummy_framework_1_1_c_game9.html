Detailed Description

Base class for DummyFramework applications.

The CGame9 class is responsible for Direct3D initialization, device loss/reset handling and fixed timestep updating of the application's logic.

Note that the Update() method is called in discrete time steps (every 1/10 th second by default), that can be modified through the Sync.UpdateInterval member. Fixed timestep updating can be turned off by setting the fixedtimestep member to false.

Another important thing is state management, that was recently introduced into the framework. This requires that if something can be called on the StateManager object then that has to be called on it. This implies that state blocks should be used with care (that means you have to guarantee that the framework stays consistent).
